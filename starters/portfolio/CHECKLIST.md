# Portfolio Launch Checklist

Also run [playbook CHECKLIST-LAUNCH](../../playbook/CHECKLIST-LAUNCH.md).

---

## Content honesty

- [ ] No fake client names or made-up case studies
- [ ] Example work labeled as concepts/styles if not real projects
- [ ] Skills list matches tools client actually uses
- [ ] Pricing matches real offer

---

## One-page sections

- [ ] Hero loads with correct name and pitch
- [ ] About section accurate
- [ ] Work/examples section updated
- [ ] Services list correct
- [ ] Contact form points to client email
- [ ] Calendly section shows only if URL set
- [ ] Social links show only if URLs set

---

## Integrations

- [ ] FormSubmit: first test submit activates inbox
- [ ] `thanks.html` redirect works
- [ ] Calendly embed loads (if used)
- [ ] Payment buttons go to correct links (if used)

---

## GitHub Pages

- [ ] Repo named `clientname-web-portfolio`
- [ ] Pages enabled (branch `main`, `/`)
- [ ] `siteUrl` in config matches live URL
- [ ] Custom domain connected (optional)
